In `app/auth/routes.py`, change the `logout` function (lines 354-358):

**FROM:**
```python
@router.post("/logout")
async def logout(request: Request, response: Response):
    response.delete_cookie("session")
    response.delete_cookie("supabase_refresh")
    return {"message": _msg(request, "auth.logged_out")}
```

**TO:**
```python
@router.post("/logout")
async def logout(request: Request):
    response = RedirectResponse(url="/auth/login", status_code=303)
    response.delete_cookie("session")
    response.delete_cookie("supabase_refresh")
    return response
```

Key changes:
- Remove `response: Response` parameter (no longer injected by FastAPI — we create our own)
- Return `RedirectResponse(url="/auth/login", status_code=303)` — 303 is the correct status for POST-redirect-GET pattern
- Cookie deletion calls remain on the response object

In `tests/test_auth.py`, add test:
```python
def test_logout_redirects_to_login(client):
    """POST /auth/logout should redirect to /auth/login, not return JSON."""
    resp = client.post("/auth/logout", allow_redirects=False)
    assert resp.status_code == 303
    assert resp.headers["location"] == "/auth/login"
```

Also add a test verifying cookies are cleared:
```python
def test_logout_clears_cookies(client):
    """POST /auth/logout should clear session and refresh cookies."""
    resp = client.post("/auth/logout", allow_redirects=False)
    cookie_headers = resp.headers.get_list("set-cookie") if hasattr(resp.headers, 'get_list') else [v for k, v in resp.headers.raw if k == b"set-cookie"]
    cookie_str = str(cookie_headers)
    assert "session" in cookie_str.lower()
```
