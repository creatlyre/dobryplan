  <action>
1. Install Capacitor packages:
   ```bash
   npm install @capacitor/core @capacitor/cli @capacitor/android
   ```

2. Create `capacitor.config.ts` at project root with these EXACT values:
   ```typescript
   import { CapacitorConfig } from '@capacitor/cli';

   const config: CapacitorConfig = {
     appId: 'app.dobryplan.twa',
     appName: 'Dobry Plan',
     webDir: 'www',
     server: {
       url: 'https://synco-production-e9da.up.railway.app',
       cleartext: false,
       errorPath: '/index.html'
     },
     android: {
       minWebViewVersion: 60,
       buildOptions: {
         keystorePath: '../dobryplan-keystore.jks',
         keystoreAlias: 'dobryplan'
       }
     }
   };

   export default config;
   ```

   Key points:
   - `appId` MUST be `app.dobryplan.twa` (same as TWA for in-place updates)
   - `server.url` points to production Railway deployment
   - `server.errorPath` is `/index.html` — Capacitor serves this from local `www/` when server unreachable
   - `webDir` is `www` — Capacitor copies this into the APK
   - `android.buildOptions.keystorePath` is relative to `android/app/` directory

3. Create `www/` directory with offline fallback. Create `www/index.html` based on `public/offline.html` but enhanced:
   - Full HTML5 document with `<meta charset="utf-8">`, viewport meta
   - Title: "Dobry Plan — Offline"
   - Inline CSS matching app brand colors (background: #0f0a2e, text: white, accent: #6c5ce7)
   - Show app name "Dobry Plan", offline icon (SVG inline), message "Brak połączenia z internetem / No internet connection"
   - A "Spróbuj ponownie / Try again" button that calls `window.location.reload()`
   - Must NOT reference external CSS/JS files (no network available)
   - Include `<head>` tag (required for Capacitor bridge injection)

4. Copy `public/css/style.css` to `www/css/style.css` (Capacitor may need it for initial load)

5. Update `.gitignore` — add these lines if not already present:
   ```
   # Capacitor
   android/app/build/
   android/.gradle/
   android/capacitor-cordova-android-plugins/
   node_modules/
   ```

   Do NOT gitignore the `android/` directory itself (it contains the project source).
   Do NOT gitignore `www/` (it contains the offline fallback).

  <action>
1. **Back up and remove** the current TWA android directory:
   ```bash
   # Git tracks the old directory, so removal is safe
   rm -rf android/
   ```

2. **Generate Capacitor Android project:**
   ```bash
   npx cap add android
   ```
   This creates a fresh `android/` directory with:
   - Capacitor's `BridgeActivity` as the main activity (NOT TWA's LauncherActivity)
   - Standard Android project structure with Capacitor dependencies
   - `android/app/src/main/assets/public/` — web assets copied from `www/`

3. **Re-apply signing configuration** to `android/app/build.gradle`. After the Capacitor-generated `android {}` block, add or modify the `signingConfigs` and `buildTypes` sections:

   Inside the `android { }` block, add:
   ```gradle
   signingConfigs {
       release {
           storeFile file('../dobryplan-keystore.jks')
           storePassword System.getenv('KEYSTORE_PASSWORD')
           keyAlias 'dobryplan'
           keyPassword System.getenv('KEY_PASSWORD')
       }
   }
   ```

   In the `buildTypes` block, update `release`:
   ```gradle
   buildTypes {
       release {
           minifyEnabled false
           proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
           signingConfig signingConfigs.release
       }
   }
   ```

4. **Update color resources** in `android/app/src/main/res/values/colors.xml` — ensure brand colors match:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <resources>
       <color name="colorPrimary">#1e1553</color>
       <color name="colorPrimaryDark">#0f0a2e</color>
       <color name="colorAccent">#6c5ce7</color>
   </resources>
   ```

5. **Update strings** in `android/app/src/main/res/values/strings.xml` — add asset_statements for DAL:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <resources>
       <string name="app_name">Dobry Plan</string>
       <string name="title_activity_main">Dobry Plan</string>
       <string name="package_name">app.dobryplan.twa</string>
       <string name="custom_url_scheme">app.dobryplan.twa</string>
       <string name="asset_statements">
           [{
               \"relation\": [\"delegate_permission/common.handle_all_urls\"],
               \"target\": {
                   \"namespace\": \"web\",
                   \"site\": \"https://synco-production-e9da.up.railway.app\"}
           }]
       </string>
   </resources>
   ```

6. **Copy app icons** to Android resource directories:
   ```bash
   # Copy launcher icon
   mkdir -p android/app/src/main/res/mipmap-xxxhdpi
   mkdir -p android/app/src/main/res/drawable
   cp public/icons/icon-192.png android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png
   cp public/icons/icon-512.png android/app/src/main/res/drawable/splash.png
   ```

7. **Verify the AndroidManifest.xml** uses Capacitor's `MainActivity` (which extends `BridgeActivity`), NOT `LauncherActivity`. The Capacitor-generated manifest should already have this — just confirm it does NOT contain `com.google.androidbrowserhelper`.

8. **Sync web assets to Android project:**
   ```bash
   npx cap sync android
   ```
   This copies `www/` contents to `android/app/src/main/assets/public/` and installs Capacitor Android dependencies.

9. **Verify build compiles locally** (if Java/Android SDK available):
   ```bash
   cd android && ./gradlew assembleDebug 2>&1 | tail -5 && cd ..
   ```
   If SDK not available locally, this will be verified in CI (Plan 02). Skip gracefully if build tools missing.
