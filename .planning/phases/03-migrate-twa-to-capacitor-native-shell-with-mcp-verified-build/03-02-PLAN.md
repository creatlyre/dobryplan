Rewrite `.github/workflows/android-build.yml` to build a Capacitor Android APK. The overall structure stays the same (trigger, permissions, Java/Android SDK setup, keystore decode, version extract, Gradle build, upload, release) but with these specific changes:

1. **Update header comment** — change from "TWA wrapper via Gradle" to "Capacitor Android app":
   ```yaml
   # Build Android APK (Capacitor native shell)
   #
   # Uses Capacitor to sync web assets, then Gradle to build the signed APK.
   ```

2. **Add path trigger** for Capacitor config:
   ```yaml
   paths:
     - 'android/**'
     - 'www/**'
     - 'capacitor.config.ts'
     - 'public/manifest.json'
     - 'public/sw.js'
     - 'public/icons/**'
     - 'pyproject.toml'
     - 'package.json'
   ```

3. **Add Node.js setup step** AFTER Java setup (before Gradle):
   ```yaml
   - name: Set up Node.js
     uses: actions/setup-node@v4
     with:
       node-version: '20'
       cache: 'npm'

   - name: Install dependencies
     run: npm ci
   ```

4. **Add Capacitor sync step** AFTER npm install and BEFORE Gradle build:
   ```yaml
   - name: Sync Capacitor web assets
     run: npx cap sync android
   ```

5. **Remove** the "Prepare Android resources" step that manually copies icons (Capacitor handles this via sync + the resources already in android/app/src/main/res/).

6. **Update Gradle build step** — use `./gradlew` wrapper instead of system `gradle`:
   ```yaml
   - name: Build release APK
     working-directory: android
     run: ./gradlew assembleRelease
     env:
       KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
       KEY_PASSWORD: ${{ secrets.KEY_PASSWORD }}
   ```

7. **APK output path stays the same**: `android/app/build/outputs/apk/release/app-release.apk`

8. **Keep version extraction** from pyproject.toml (unchanged).

9. **Keep GitHub Release step** (unchanged — uses softprops/action-gh-release@v2).

10. **Remove** any Bubblewrap references (there shouldn't be any since Phase 1 already used Gradle, but verify).

Complete workflow should have these steps in order:
1. Checkout
2. Setup Java 17
3. Setup Android SDK
4. Accept SDK licenses
5. Install SDK components
6. Setup Gradle
7. **Setup Node.js 20** (NEW)
8. **npm ci** (NEW)
9. Decode keystore
10. Extract version
11. **npx cap sync android** (NEW — replaces old "Prepare Android resources")
12. Build release APK (use `./gradlew` not `gradle`)
13. Upload APK artifact
14. Create GitHub Release
