**2a. SupabaseStore connection pooling:**

Modify `SupabaseStore` to accept an optional `httpx.Client` in `__init__` and reuse it across all operations instead of creating `with httpx.Client()` per query.

Create a module-level client factory:
```python
_shared_client: httpx.Client | None = None

def _get_shared_client() -> httpx.Client:
    global _shared_client
    if _shared_client is None or _shared_client.is_closed:
        _shared_client = httpx.Client(timeout=20.0)
    return _shared_client
```

In each method (`select`, `insert`, `update`, `delete`, `count`), replace:
```python
with httpx.Client(timeout=20.0) as client:
    response = client.get(...)
```
With:
```python
client = self._client
response = client.get(...)
```

Store `self._client = _get_shared_client()` in `__init__`.

**2b. Static file serving in main.py:**

Add to main.py:
```python
from fastapi.staticfiles import StaticFiles
app.mount("/static", StaticFiles(directory="public"), name="static")
```

Place this BEFORE the route registrations so static files are served efficiently. Add `headers` parameter for cache control if supported, or use a custom middleware for static Cache-Control.

Since FastAPI's StaticFiles doesn't natively support custom Cache-Control, add a simple approach: configure StaticFiles normally. The browser will cache based on Last-Modified/ETag which StaticFiles provides automatically.

**2c. Ensure `public/css/` directory exists:**

Create `public/css/` directory if it doesn't exist.
