<objective>
Fix Method Not Allowed (405) errors caused by 307 redirects preserving HTTP methods, and eliminate browser confirmation dialogs on public pages by converting the logout form to a GET link and replacing native alert() calls with inline toasts.

Purpose: Users hitting protected routes without auth get clean redirects instead of 405 errors. Logout no longer triggers "Confirm Form Resubmission" on browser Back. Pricing page errors use polished inline UI instead of ugly native dialogs.

Output: 4 fixed production files + 2 updated test files
