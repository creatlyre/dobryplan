# Phase 12 — UI Design Contract

> Visual and interaction contract for the Budget Settings page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom glass morphism) |
| Preset | not applicable |
| Component library | none — Jinja2 templates + HTMX |
| Icon library | Unicode emoji (consistent with existing nav) |
| Font | System default (Tailwind `font-sans` — no custom fonts loaded) |

**Note:** This project uses a custom glass morphism design system defined in `app/templates/base.html` `<style>` block. All new UI must use the existing `.glass`, `.glass-btn-*`, `.glass-input` classes. No new CSS classes should be introduced unless strictly necessary.

---

## Spacing Scale

Declared values (Tailwind utility classes — multiples of 4):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2`, `gap-2` | Compact element spacing, button groups |
| md | 16px | `p-4`, `gap-4` | Default element spacing, form field gaps |
| lg | 24px | `p-6`, `gap-6` | Section padding, card body |
| xl | 32px | `p-8`, `gap-8` | Layout gaps between major sections |

Exceptions: None — follow existing `px-4 py-6` main container pattern from `base.html`.

## Typography

| Role | Size | Tailwind | Weight | Line Height |
|------|------|----------|--------|-------------|
| Body | 14px | `text-sm` | 400 (`font-normal`) | 1.5 (`leading-normal`) |
| Label | 14px | `text-sm` | 600 (`font-semibold`) | 1.5 |
| Heading (section) | 18px | `text-lg` | 600 (`font-semibold`) | 1.3 |
| Page title | 20px | `text-xl` | 600 (`font-semibold`) | 1.2 |

Two weights only: 400 (body text, input values) and 600 (labels, headings, titles). Existing nav bar uses `font-bold` (700) but that is in `base.html` and not modified by this phase.

### Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| `lg` (1024px+) | 2-column: sidebar (w-48/w-56) + main content area |
| Below `lg` | Single column: sidebar becomes horizontal nav row above form; back button in top-left |

### Sidebar Specs

- Width: `w-48` (192px) on desktop
- Glass panel: `.glass rounded-2xl`
- Nav items: `px-4 py-3 text-sm font-medium rounded-xl` with hover `bg-white/10` (dark) / `bg-indigo-100/50` (light)
- Active item: `glass-btn-info` style (blue-tinted background)
- Back link: bottom of sidebar, `glass-btn-secondary` styled, `← Kalendarz` / `← Calendar`
- Future sections (Income, Expenses, Overview): shown as disabled/muted items in sidebar, not clickable yet

### Form Sections

Each section is a `.glass rounded-2xl p-6` card with:
- Section heading: `text-lg font-semibold mb-4`
- Fields stacked with `space-y-4` gap
- Input: `.glass-input rounded-xl` (existing class)
- Label: `text-sm font-medium mb-1 block`
- Currency suffix "PLN" as inline text right of input (use `flex items-center gap-2` row)

## Input Behavior

| Field | Type | Placeholder | Validation | Default |
|-------|------|-------------|------------|---------|
| Rate 1 | `number` | `0.00` | > 0, max 2 decimals | empty |
| Rate 2 | `number` | `0.00` | > 0, max 2 decimals | empty |
| Rate 3 | `number` | `0.00` | > 0, max 2 decimals | empty |
| ZUS + Accounting | `number` | `0.00` | > 0, max 2 decimals | empty |
| Bank Balance | `number` | `0.00` | ≥ 0, max 2 decimals | empty |

- All inputs: `step="0.01"`, `min="0"` HTML attributes
- Bank Balance allows 0 (zero starting balance valid); rates and costs must be > 0
- Inline validation on blur: red border (`border-red-400`) + error text below field
- Valid state: default border (no green — keep it clean)

## Interaction States

| State | Visual Treatment |
|-------|-----------------|
| Page loading | Skeleton: 3 glass panels with pulsing `animate-pulse bg-white/5` blocks |
| Empty (first visit) | Empty state card with heading + body text + CTA pointing to form fields |
| Form filled | All sections showing current values, Save button enabled |
| Saving | Save button shows spinner (CSS `animate-spin` on small circle) + "Zapisuję..." / "Saving..." text, disabled |
| Save success | Brief green toast/banner at top of form: "✓ Ustawienia zapisane" — auto-dismiss after 3 seconds |
| Validation error | Red border on invalid field + error text below, Save button disabled |
