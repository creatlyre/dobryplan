**API Tests (using mock SupabaseStore):**

1. `test_get_budget_settings_empty` — GET /api/budget/settings when no settings exist → returns `{"data": null}`. Mock `db.select("budget_settings", ...)` to return `[]`.

2. `test_get_budget_settings_existing` — GET /api/budget/settings when settings exist → returns `{"data": {rate_1: 150.0, ...}}`. Mock `db.select("budget_settings", ...)` to return a row dict with all fields.

3. `test_save_budget_settings_create` — PUT /api/budget/settings with valid payload when no settings exist → returns 200 with saved data. Mock `db.select` returning `[]` then `db.insert` returning the new row.

4. `test_save_budget_settings_update` — PUT /api/budget/settings with valid payload when settings exist → returns 200 with updated data. Mock `db.select` returning existing row, then `db.update` returning updated row.

5. `test_save_budget_settings_validation_negative_rate` — PUT /api/budget/settings with `rate_1: -10` → returns 422 validation error. No mock needed (Pydantic validates before route logic).

6. `test_save_budget_settings_validation_negative_balance` — PUT /api/budget/settings with `initial_balance: -100` → returns 422.

7. `test_save_budget_settings_validation_zero_rate` — PUT /api/budget/settings with `rate_1: 0` → returns 422 (must be > 0).

8. `test_save_budget_settings_zero_balance_allowed` — PUT /api/budget/settings with `initial_balance: 0.0` and valid rates → returns 200 (zero balance is valid).

9. `test_save_budget_settings_rounds_to_two_decimals` — PUT with `rate_1: 150.999` → response has `rate_1: 151.0` (rounded to 2 decimals).

**View Tests:**

10. `test_budget_settings_page_renders` — GET /budget/settings → returns 200 HTML containing `budget.section_rates` translated text (Polish default, or at least the template marker).

11. `test_budget_settings_page_has_nav_link` — GET / (home page) → response HTML contains `/budget/settings` link.

12. `test_budget_settings_page_unauthenticated` — GET /budget/settings without auth → redirects or returns 401.

**Test fixture pattern:** Follow existing conftest.py patterns for creating authenticated client with mocked SupabaseStore. Use `unittest.mock.patch` or the existing mock_db fixture to intercept database calls.

Each test should be independent — mock fresh per test, no shared mutable state.
